# 转写与纪要主流程 (/meeting-minutes)

## 1. 编排器参数与约定

| 参数 | 说明 |
| --- | --- |
| `Final_Output` | 最终输出的纪要文件 |
| `Rule_Root` | 规则预加载路径，优先读取 `rules/minutes-rules.md` 和 `rules/minutes-template.md` |
| `LongDoc_Mode` | 文档超过 35k tokens（约 2 万中文字符）时启用 |
| `Meeting_Type` | `cross_dept`：跨部门/外部对接会；`internal`：内部工作会/任务分工会 |
| `Template_Chapters` | 从模板加载的章节清单（7 个定义章节） |

## 2. 执行顺序

1. **输入解析与预读**：按扩展名执行路由（音频走 Whisper，文档走对应 Skill，文本直读），静默加载全部规则。
2. **会议类型判定**：
   - 含 2 个及以上外部单位/机构 → `cross_dept`
   - 仅内部团队参与，以工作安排/排期为主 → `internal`
   - 无法判定时默认 `cross_dept`
3. **状态回执**：音频处理时输出 `正在使用本地 Whisper 处理录音...`；转写完成输出 `转写完成，开始提炼纪要...`。
4. **信息提取与澄清**：按类型提取元信息（跨部门：参会单位/参会人/会议主题；内部：主持人/与会人员/记录人）。关键背景严重缺失时最多提 3 个澄清问题。
5. **长文分段提炼**：按议题切段，每段提炼后清掉原文，仅保留核心结论。
6. **全局合并与清洗**：去重、统一实体名、化解冲突。一个议题只说一次，将需求、共识、决策、技术要点合并到同一议题下。
7. **章节过滤与动态编号**：按 `Meeting_Type` 过滤章节清单，保留的章节按顺序重新编号。必选章节正常输出，可选章节若不涉及保留标题写 `**本次不适用**`。输出标题必须干净。
8. **最终出稿**：按过滤后的章节顺序生成终稿，≤ 5 章。
9. **自动质检**：出稿后触发 `/纪要核验`，结果附在纪要末尾。

## 3. 章节过滤速查

### 跨部门对接会 → 5 章

| 输出序号 | 章节 | 性质 |
| :---: | --- | :---: |
| 一 | 会议背景 | 必选 |
| 二 | 意见总结 | 必选 |
| 三 | 修改方案 | 可选 |
| 四 | 待办事项 | 必选 |
| 五 | 后续行动 | 必选 |

### 内部工作会 → 5 章

| 输出序号 | 章节 | 性质 |
| :---: | --- | :---: |
| 一 | 会议目标 | 必选 |
| 二 | 意见总结 | 必选 |
| 三 | 详细排期表 | 可选 |
| 四 | 待办事项 | 必选 |
| 五 | 后续行动 | 必选 |

## 4. 关键写作约束

- **去重红线**：意见总结是唯一的事实出口，每个议题的需求、共识、决策、技术要点集中闭环。
- **背景精简**：≤ 3 句话。
- **意见总结 vs 修改方案分工**：前者说"共识结论"，后者说"落实到文档的改法"，两者不重复。
