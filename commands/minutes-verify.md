# 纪要质量核验 (/纪要核验)

本命令作为末端质量闸门，**仅检查、不生成新内容**，发现问题时指导出稿命令修正。

> 必须同时提供纪要文件与原始文本，否则无法执行事实核查。若原始文本未提供，向用户索要。

## 1. 事实核查

- 遍历意见总结章节中的每条决策与修改方案章节中的每条修改要求，在原始文本中定位依据。
- **拦截动作**：无依据或明显属于模型主观推测的，标记 `[需删除/降级为待确认]`。

## 2. 结构完整性

- 按模板章节清单及当前会议类型，检查必选章节是否齐全。
- 检查可选章节若不涉及是否正确标注 `**本次不适用**`。
- 检查意见总结中每条决策是否附带证据锚点 —— 跨部门会议要求 `状态 + 原文快照或时间戳`，内部工作会要求 `负责人`。
- **拦截动作**：章节缺失或证据格式不符，标记 `[结构/格式错误]`。

## 3. 行动项完整性

- 待办事项表：是否逐项包含负责人、截止日期、交付物、优先级（跨部门会议另需验收标准）。
- 后续行动：是否明确了待决事项的下一步、下一轮节点或里程碑、风险跟踪或跟进机制。
- **拦截动作**：任务边界模糊或缺少责任主体，标记 `[表述含糊-需澄清]`。

## 4. 核验输出模板

```markdown
## 质量核验结果

- **会议类型：** [跨部门对接会 / 内部工作会]
- **总体结论：** [通过 / 基本合规 / 需修改]

### 待修正问题清单
1. [高风险/中风险/低风险] 问题描述
   - 位置：[对应章节]
   - 修正建议：[具体修正动作]

### 发布判定
- 判定标准：
  - **通过**：必选章节齐全，低风险问题 ≤ 2 个。
  - **基本合规**：必选章节齐全，但有 3 个以上格式问题。
  - **需修改**：缺失必选章节，或存在 2 个及以上高风险项（如会议基本要素缺失、关键结论无依据）。
- **是否可直接发布：** [是 / 否]
- **需修复的核心问题数：** [N]
```
