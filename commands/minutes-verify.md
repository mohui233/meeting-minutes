# 纪要质量核验 (/纪要核验)

本命令作为末端质量关口，**仅检查、不生成新内容**，并在发现问题时指导出稿命令进行修正。

> **注意**：必须同时提供“纪要文件”与“原始文本”，否则无法执行防幻觉事实核查。若原始文本未提供，请向用户索要。

## 1. 事实核对（防幻觉）
- 检查【修改方案】与【后续行动】：在“原始文本”中是否能找到依据？
- **拦截动作**：若无依据或明显属于模型的主观推测，标记为 `[需删除/降级为待确认]`。

## 2. 结构与定位检查
- 检查大纲：是否完整包含了模板要求的所有一级标题（包括标注为“本次不适用”的模块）？
- 检查定位契约：【关键决策】是否附带了合规的原文快照或时间戳？
- **拦截动作**：若结构缺失、私自隐藏标题或缺失证据定位，标记为 `[结构/格式错误]`。

## 3. 完整性检查
- 检查【修改方案】：是否清晰界定了各单位的具体任务与修改要求？
- 检查【后续行动】：是否明确了牵头单位/责任人以及时间节点？
- **拦截动作**：若任务边界模糊或缺少责任主体，标记为 `[表述含糊-需澄清]`。

## 4. 核验输出模板
```markdown
## 质量核验结果
- 总体结论：[通过 / 基本合规 / 需修改]

## 待修正问题清单
1. [高风险/中风险/低风险] 问题描述
   - 位置：[对应章节]
   - 修正建议：[例如：删除无证据的推断 / 补充后续行动的责任单位 / 恢复被隐藏的必选标题]

## 发布判定
- 判定标准：
  - **通过**：必填项全，低风险问题 ≤ 2 个。
  - **基本合规**：必填项全，但有 3 个以上格式问题。
  - **需修改**：缺失必填项或存在 2 个及以上高风险项（如会议时间、结论缺失）。
- 是否可直接发布：[是 / 否]
- 需修复的核心问题数：[N]
```
