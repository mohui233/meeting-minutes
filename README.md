# 会议纪要 Skill (meeting-minutes)

将会议录音或文本快速转化为符合政务/业务标准的结构化会议纪要，采用"分层编排 + 证据约束 + 统一模板动态过滤"设计，内置防幻觉质检闸门，跨部门对接会/内部工作会均≤5章。

## 设计思路

以前手动整理纪要很耗时间，录音反复听，重点容易漏。后来直接把转写稿丢给 AI 出纪要，看着完整，但人就在现场就知道哪里不对。于是本 Skill 围绕三个核心原则设计：

### 1. 先定骨架，别让 AI 自由发挥

统一模板管住输出结构，按会议类型（跨部门/内部）自动过滤章节后出稿，每种类型≤5章。标题不能删，不涉及就写"本次不适用"。转写稿走固定流程——切段、提炼、合并，不让模型自己猜结构。会议背景压到 3 句话以内。

### 2. 先捋要点，再出稿，同一件事不说两遍

不让模型一上来直接写终稿。长会先按议题切段，每段提炼核心信息后清掉原文，只保留结构化要点。最关键的是：**需求、共识、决策、技术要点在"意见总结"中一次性闭环**，不拆分到多个章节重复表述。最终按类型过滤章节、动态编号出稿。

### 3. 拿证据卡住，再用质检兜底

定了条死规矩：所有"共识""方案""行动项"，都必须能在原话里找到依据；找不到就写"待确认"。每个关键决策带上状态和原文依据。

光写规则还不够，模型不一定真听。所以又加了一步自动质检——仅检查不补写，分三个维度：决策能否回指原文、必选章节是否齐全、行动项是否具备责任人和截止日。

## 文件结构

```
meeting-minutes/
├── SKILL.md                          # 技能入口：元信息、触发方式、路由策略、交付约定
├── README.md                         # 本文件
├── commands/
│   ├── meeting-minutes.md            # 主编排器：路由调度、分段提炼、上下文清理、出稿
│   └── minutes-verify.md             # 质量闸门：事实核对、结构检查、完整性检查
└── rules/
    ├── minutes-rules.md              # 规则知识库：底线约束、会议类型判定、行文规范与执行红线
    └── minutes-template.md           # 统一输出大纲，按类型过滤后≤5章
```

## 核心能力

- **精简纪要**：跨部门对接会/内部工作会均≤5章，合并重复表述，一个议题只说一次
- **防幻觉机制**：所有结论必须能在原文中找到依据，找不到就写"待确认"；关键决策附带状态与原文快照
- **自动质检**：出稿后自动触发核验，检查事实依据、结构完整性、行动项责任人/截止日，判定可否直接发布
- **长文分段**：超长会议自动按议题切段提炼，避免上下文溢出
- **实体消歧**：分段提炼后全局合并去重，统一实体名称，化解信息冲突
- **双模路由**：自动判定跨部门对接会/内部工作会，匹配不同元信息字段和章节组合
- **最终交付**：输出 `<原文件名>-会议纪要.md`，质检结果以注释形式附在纪要末尾
